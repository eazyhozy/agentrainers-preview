# 레거시 재고 관리 시스템 리팩토링

## 개요

500줄이 넘는 레거시 재고 관리 모듈을 리팩토링하세요. 모든 테스트가 리팩토링 전후로 통과해야 합니다.

`src/inventory.py` 파일에는 여러 안티패턴이 포함된 레거시 코드가 있습니다. 코드의 외부 동작을 변경하지 않으면서 내부 구조를 개선하는 것이 목표입니다.

## 규칙

- **기능 변경 금지**: 모든 공개 함수의 입출력이 동일해야 합니다.
- **새로운 의존성 추가 금지**: 외부 패키지를 추가할 수 없습니다.
- **테스트 파일 수정 금지**: `tests/` 디렉토리의 파일을 수정할 수 없습니다.

## 리팩토링 품질 목표

다음은 리팩토링 시 참고할 가이드라인입니다:

- **함수 길이**: 각 함수는 30줄 이하로 유지
- **중복 코드 제거**: 반복되는 로직을 공통 함수로 추출
- **매직 넘버 제거**: 의미 없는 숫자 리터럴을 명명된 상수로 교체
- **전역 상태 제거**: 모듈 레벨 가변 상태를 클래스로 캡슐화
- **타입 힌트 추가**: 모든 함수에 파라미터와 반환 타입 명시
- **적절한 클래스 분리**: 관련 데이터와 동작을 클래스로 그룹화
- **의미 있는 변수/함수 이름**: 약어 대신 명확한 이름 사용

## 풀이 가이드

- `tests/test_public.py` — 공개 테스트입니다. 풀이 중 자유롭게 참고하세요.
- `tests/test_hidden.py` — 최종 채점용 엣지케이스 테스트입니다. 풀이 과정에서는 참고하지 않고, 코드 품질 개선에 집중해 보세요.

## 테스트 실행

```bash
docker compose run --rm runner
```
